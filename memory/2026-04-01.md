# 2026-04-01

## AI SDK Migration (useChat)
- **Migrated chat page** from raw `fetch` + manual state management → `useChat` hook + `DefaultChatTransport`
- **Route.ts** now translates backend JSON-lines events to AI SDK UIMessageStream protocol:
  - `content` → `text-start` / `text-delta` / `text-end`
  - `tool_call` → `tool-input-start` + `tool-input-available`
  - `tool_result` → source extraction (source-url events)
  - `error` → text-delta with warning
- Auth token passed via body (`_auth_token`) since Next.js API routes don't expose custom headers
- Sources extracted from web search tool results and emitted as `source-url` stream parts
- Sources also parsed from message parts on the frontend (dual extraction: parts + top-level sources field)
- Tool rendering uses typed cast for `state`, `input`, `output` from tool parts
- Same Seedify dark theme UI preserved: tool cards with icons/spinners, sources panel, thinking indicator, error state
- `experimental_throttle: 50` for smooth streaming
- Build passes, committed `09b0a35`, pushed to GitHub

### Key changes
- `src/app/api/chat/route.ts` — 130 → ~160 lines, now handles all 4 backend event types
- `src/app/chat/page.tsx` — 290 → ~310 lines, removed manual fetch/streaming, uses useChat
- No new dependencies (ai v6 + @ai-sdk/react v3 already installed)

### Known limitations
- Tool parts stay in `input-available` state (no `tool-output-available` event) because backend sends tool_call and tool_result as separate events without shared IDs. This is cosmetic — tools still show as done with green checkmark.
- Vercel deploy will auto-trigger from push
